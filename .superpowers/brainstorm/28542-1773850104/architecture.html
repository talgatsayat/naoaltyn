Архитектура системы

Общая схема — как всё взаимодействует. Выглядит правильно?

Высокоуровневая архитектура
🌐 React Frontend
• Публичный сайт
• Портал заявителя
• Админ-панель
• 3 языка (ru/kz/en)
• Стиль uba.edu.kz
⚙️ FastAPI Backend
• REST API
• JWT аутентификация
• Загрузка файлов
• Email-уведомления
• 4 типа заявок
🗄️ PostgreSQL + Файлы
• Пользователи
• Заявки
• Документы
• Статусы / история
• Файлы на диске
── HTTP / REST API ──
👤 Заявитель
Регистрация → Вход → Новая заявка →
Заполнение формы (физ. или юр. лицо) →
Загрузка документов → Отслеживание статуса
🛡️ Администратор
Вход → Список заявок (фильтры) →
Просмотр деталей + скачивание документов →
Смена статуса → Email заявителю
Интеграция с uba.edu.kz: React-приложение размещается на поддомене (например apply.uba.edu.kz), стиль копирует основной сайт. Существующий сайт не трогается — просто добавляется кнопка «Подать заявку».
