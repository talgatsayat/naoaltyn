Как будем строить систему?

3 подхода к технической реализации — выберите один

Next.js + PostgreSQL + Файлы локально
✅ Один проект — и сайт, и API, и база
✅ Быстро развернуть, легко расширять
✅ Подходит для тестовой и боевой версии
⚡ 3 языка через next-intl
📁 Файлы хранятся на сервере (позже → S3)

A — Next.js монолит ⭐ Рекомендую

Современный фреймворк, всё в одном проекте. Лучший выбор для MVP и дальнейшего роста.

React + FastAPI (Python) + PostgreSQL
✅ Отдельный бэкенд — легче интегрировать с uba.edu.kz
✅ Python удобен если есть Python-разработчики
⚠️ Два отдельных сервиса — сложнее деплой
⚠️ Дольше настраивать

B — React + FastAPI (раздельный)

Фронтенд и бэкенд отдельно. Хорошо если команда на Python.

PHP / Laravel + MySQL
✅ Легко хостить на любом дешёвом сервере
✅ Многие казахстанские сайты на PHP
⚠️ Устаревший подход
⚠️ Хуже для 3-языкового интерфейса

C — Laravel + MySQL (классика)

Традиционный подход, простой хостинг на shared-сервере.
